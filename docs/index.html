Envoy

Environment setup, handled.

npx @allons-y/envoy

The problem

Every project starts the same way: copy .env.example → .env, then hunt through Notion, 1Password, Slack history, or your own memory for the actual values. In a monorepo it's worse — five packages, five example files, the same ritual repeated for each one.


If you keep a root ~/.env with your real values, Envoy bridges the gap. It reads your .env.example as a template, pulls matching keys from ~/.env, and writes a complete .env alongside it — preserving every comment and blank line in the process.

Features

Template-driven

.env.example defines the shape; ~/.env supplies the values.

Safe by default

Skips any .env that already exists. Use --force to overwrite.

Security-first

Refuses to write secrets into a git-tracked file. Warns if .env isn't gitignored.

Monorepo-aware

Recursively finds every .env.example under your project root, skipping node_modules.

Non-destructive

Keys absent from ~/.env fall back to the example value — nothing is lost.

Preview mode

--dry-run shows exactly what would be written without touching the filesystem.

MCP tool

Expose copy_env to any MCP-compatible host — Claude Desktop, Claude Code, and more.

No magic

No network calls. No config files. No global state. Small, readable, fully tested.

How it works

Scans your directory recursively for .env.example files, ignoring node_modules.

For each one, checks whether a .env already exists alongside it — skips unless --force.

Runs git safety checks — blocks if .env is tracked; warns if it isn't gitignored.

Reads ~/.env (or --root) into a key → value map.

Walks every line in .env.example: comments and blanks pass through unchanged; matching keys get your real value substituted; missing keys fall back to the example value.

Writes the result to .env next to the example file.

CLI reference

Run in any project root. Envoy finds every .env.example recursively and creates the corresponding .env.

envoy [options]

Flag Alias Description
--force -f Overwrite existing .env files
--dry-run -n Preview changes without writing any files
--root <path> -r Custom root env file (default: ~/.env)
--dir <path> -d Directory to scan (default: current directory)
--skip-audit -s Skip the git safety checks
--help -h Show help 

# Preview what would be created, without writing anything
envoy --dry-run

# Regenerate .env files from scratch
envoy --force

# Use a team-shared env file instead of ~/.env
envoy --root ./secrets/.env.shared

# Scan a specific subdirectory
envoy --dir packages/api

MCP tool

Envoy ships an MCP server so AI tools can call copy_env directly. The tool accepts the same options as the CLI: dir, force, dry_run, root_env_path, and skip_audit.

Claude Desktop

{
  "mcpServers": {
    "envoy": {
      "command": "npx",
      "args": ["-y", "@allons-y/envoy/mcp"]

What's next

Envoy is intentionally minimal today — it does one thing and does it well. The bigger vision is to make it the single tool that bridges your personal secrets and every project you work in, without friction or foot-guns. That means smarter value handling, better control over what gets scanned and where, and tighter integration with the safety checks developers already rely on.
